# Jarvis Codex Gate 2 Prototype Flow

## Local Review Surface

```text
+----------------------------------------------------------+
| Jarvis Plan                                              |
| Local-only review surface served from 127.0.0.1          |
+----------------------+-----------------------------------+
| plan.mdx             | Gate 2 Stabilization              |
| canvas.mdx           | - Baseline reconciliation          |
| prototype.mdx        | - Local viewer acceptance          |
|                      | - Worktrunk lane inventory         |
|                      | - Voice notification policy        |
+----------------------+-----------------------------------+
```

## Notification Flow

```text
+----------------------------------------------------------+
| User prompt submitted                                    |
+----------------------------------------------------------+
| classify prompt: planning, UI/browser, swarm, Worktrunk, |
| hardware, GitHub, coding, or general                     |
| speak one concise phrase through local Piper when enabled|
| write routing and speech metadata to local hook logs     |
+----------------------------------------------------------+

+----------------------------------------------------------+
| Codex turn completed                                     |
+----------------------------------------------------------+
| desktop notify if duration threshold or action needed    |
| speak only when error, blocked, approval, or input needed|
| log category, speech policy, and delivery attempt        |
+----------------------------------------------------------+
```

## Worktrunk Refresh Flow

```text
+----------------------------------------------------------+
| Human-approved refresh only                              |
+----------------------------------------------------------+
| 1. Record git status and worktree list                   |
| 2. Inspect lane branch and working tree                  |
| 3. Decide refresh, abandon, or hold                      |
| 4. Rebase/merge/delete only after explicit approval      |
| 5. Return the structured worker contract                 |
+----------------------------------------------------------+
```

## Expected Interaction

Gate 2 keeps the first useful surface operational: local plan review, local state checks, and concise notifications. It avoids hosted plan publishing, background branch mutation, and noisy completion speech while leaving clear acceptance checks for future feature lanes.
